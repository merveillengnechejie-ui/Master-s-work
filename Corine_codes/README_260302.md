# Corine_codes : Scripts et Inputs ORCA pour le Projet BODIPY

## 📋 Vue d'ensemble

Ce dossier contient tous les **scripts SLURM**, **inputs ORCA 6.1**, et **fichiers de géométrie** nécessaires pour le projet de Master 2 sur l'optimisation computationnelle de nanoparticules BODIPY pour une thérapie combinée PDT/PTT.

**Portée révisée (15/11/2025)** :
- **1 molécule de référence expérimentale** (externe, publiée) pour benchmarking uniquement (pas d'évaluation finale)
- **2 prototypes internes** : Iodo-BODIPY (PDT) + TPP–Iodo–BODIPY (théranostique) - objets des calculs complets
- **Méthodologie** : ΔDFT+SOC (remplace NEVPT2)
- **Durée** : 14 semaines

## 📁 Structure des fichiers

### 1. Inputs ORCA 6.1 (Phase par phase)

#### Phase 1 : Optimisation S₀ (État fondamental)

| Fichier | Objectif | Temps estimé | Notes |
| :--- | :--- | :--- | :--- |
| `S0_gas_opt.inp` | Optimisation S₀ en phase gaz | 30–60 min | Étape de reconnaissance rapide |
| `S0_water_opt.inp` | Optimisation S₀ en milieu biologique complexe (SMD mixed) | 45–90 min | Point de départ pour tous les calculs |

**Utilisation** :
```bash
# Lancer S0 gaz
orca S0_gas_opt.inp > S0_gas_opt.out &

# Lancer S0 eau (après S0 gaz)
orca S0_water_opt.inp > S0_water_opt.out &
```

#### Phase 2 : Excitations verticales (TD-DFT)

| Fichier | Objectif | Temps estimé | Notes |
| :--- | :--- | :--- | :--- |
| `TDDFT_vertical.inp` | Calcul λ_max (absorption verticale) | 15–30 min | **Rapide et adapté pour un ordinateur (4 cœurs, 16 Go RAM)** |

**Utilisation** :
```bash
# Lancer TD-DFT (wB97X-D3) pour une approximation rapide du spectre d'absorption
orca TDDFT_vertical.inp > TDDFT_vertical.out &
```

**⚠️ Important** : En semaine 3, testez **def2-SVP vs def2-TZVP** si vous avez besoin de plus de précision. Tous les calculs seront effectués dans un milieu biologique complexe (SMD mixed).

#### Phase 3 : États excités relaxés (T₁ et S₁)

| Fichier | Objectif | Temps estimé | Notes |
| :--- | :--- | :--- | :--- |
| `T1_opt_UKS.inp` | Optimisation T₁ (état triplet) | 60–120 min | Robuste, généralement bon |
| `S1_opt_DeltaUKS.inp` | Optimisation S₁ (état singulet excité) | 120–180 min | **Étape critique** : prévoir 3–5 tentatives |

**Utilisation** :
```bash
# Lancer T1 (robuste)
orca T1_opt_UKS.inp > T1_opt_UKS.out &

# Lancer S1 (avec pré-test des guesses)
./gen_s1_guesses.sh -t S1_opt_DeltaUKS.inp -x S0_water_opt.xyz -g S0_water_opt.gbw -n 4
orca S1_opt_DeltaUKS.inp > S1_opt_DeltaUKS.out &
```

#### Phase 4 : Couplage spin-orbite (SOC)

| Fichier | Objectif | Temps estimé | Notes |
| :--- | :--- | :--- | :--- |
| `DeltaSCF_SOC.inp` | **ΔDFT+SOC** (recommandé) | 30–60 min | Cohérent avec ΔDFT, 10× plus rapide que NEVPT2 |
| `TDDFT_SOC_quick.inp` | TD-DFT SOC (Plan B) | 15–30 min | À utiliser si ΔSCF S₁ échoue |

**Utilisation** :
```bash
# Lancer ΔDFT+SOC (standard)
orca DeltaSCF_SOC.inp > DeltaSCF_SOC.out &

# Ou TD-DFT SOC (Plan B)
orca TDDFT_SOC_quick.inp > TDDFT_SOC_quick.out &
```

### 2. Exécution Locale Séquentielle (Laptop)

Les environnements HPC via SLURM ne sont plus utilisés dans cette itération; tout s'exécute directement et localement pour préserver les ressources avec une limite empirique globale par job.
Nous limitons ainsi l'usage de `%maxcore 3500` (pour 16 Go RAM) et utilisons `%pal nprocs 4`.

| Fichier | Utilité | Temps d'exécution | Notes |
| :--- | :--- | :--- | :--- |

**Utilisation générale** :
```bash
# Lancer toute la suite sur un prototype
./prepare_and_submit.sh . 4

# Surveiller
tail -f <prototype>/S0_gas_opt.out
```

## 🚀 Workflow Recommandé

### Semaine 1 : Validation de la chaîne

```bash
# 1. Tester S0 gaz sur benzène (molécule test)
orca S0_gas_opt.inp > S0_gas_opt.out

# 2. Vérifier convergence
grep "FINAL SINGLE POINT ENERGY" S0_gas_opt.out
```

### Semaine 3 : Test comparatif def2-SVP vs def2-TZVP

```bash
# 1. Utiliser le script TDDFT_vertical.inp
# 2. Vous pouvez manuellement le créer en _SVP et _TZVP et le comparer :
orca TDDFT_vertical_SVP.inp > SVP.out &
orca TDDFT_vertical_TZVP.inp > TZVP.out &

# 3. Comparer λ_max calculé vs expérimental
# Décision : Si écart < 5 nm → garder def2-SVP; si > 10 nm → garder def2-TZVP
```

### Semaines 4–6 : Calculs S₀ et TD-DFT

```bash
# 1. Lancer la boucle automatique :
./prepare_and_submit.sh . 4

# La boucle lance S0 (gaz/eau), TDDFT, T1, et SOC de façon séquentiellement asynchrone pour chaque prototype.
```

### Semaines 7–9 : Gestion de S₁ et SOC (États Excités Restants)

```bash
# 1. Entrer dans le répertoire du prototype
cd Iodo_BODIPY/

# 2. Pré-test des guesses S₁ pour les 2 prototypes (semaine 7)
../Corine_codes/gen_s1_guesses.sh -t S1_opt_DeltaUKS.inp -x S0_water_opt.xyz -g S0_water_opt.gbw -n 4

# 3. Si S₁ ne converge pas :
../Corine_codes/run_troubleshoot_S1.sh -i S1_opt_DeltaUKS.inp -x S0_water_opt.xyz -g S0_water_opt.gbw -n 4
```

## ⚠️ Troubleshooting

### Problème : ORCA manque de RAM

**Symptôme** : Erreur "out of memory" (Hardware = 16 Go)

**Solutions** :
1. Réduire `%maxcore` dans tous les `.inp` à ~`3000` (soit 12 Go de pris max pour 4 cœurs).
2. S'assurer de n'exécuter qu'UN SEUL script `orca` à la fois.
3. Réduire à la base (def2-SVP).

### Problème : S₁ ne converge pas

**Symptôme** : Énergie oscille ou augmente

**Solutions** (voir section 5 du document principal) :
1. Augmenter amortissement SCF (`DampPercentage 60`)
2. Utiliser `DIIS_TRAH` avec `TRAH_MaxDim 20`
3. Réduire pas géométrique (`MaxStep 0.1`)
4. Générer 3 guesses différents (HOMO→LUMO, HOMO−1→LUMO, HOMO→LUMO+1)
5. Utiliser `./run_troubleshoot_S1.sh` pour escalade auto

**Plan B** : Si après 5 tentatives S₁ échoue, utiliser `TDDFT_SOC_quick.inp` pour excitations verticales diagnostiques.
